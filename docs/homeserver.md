# 크롤링 파이프라인 아키텍처

## 전체 구조 개요

이 아키텍처는 **홈서버(고정 IP)** 에서 크롤링을 실행하고, 그 결과를 **AWS** 클라우드로 전달하여 후처리까지 자동화하는 파이프라인입니다.

---

## 1. 홈서버 영역 (고정 IP)

크롤링 작업이 실제로 수행되는 영역입니다.

### 구성 요소

| 컴포넌트 | 역할 |
|---|---|
| **Airflow Worker** | 크롤링 DAG 실행 — 전체 크롤링 작업을 스케줄링하고 오케스트레이션 |
| **크롤러** (Selenium / requests) | 실제 웹 크롤링 수행 |
| **S3 업로드** | 크롤링 결과를 S3 버킷에 저장 |

### 실행 흐름

```
Airflow Worker
    └─▶ 크롤러 (Selenium / requests)
            └─▶ S3 업로드 (크롤링 결과 저장)
                    ├─▶ [결과 저장] S3 버킷 (AWS)
                    └─▶ [완료 메시지] SQS 큐 (AWS)
```

---

## 2. AWS 영역

크롤링 완료 후 결과를 수신하고 후처리를 트리거하는 영역입니다.

### 구성 요소

| 컴포넌트 | 역할 |
|---|---|
| **SQS** | 크롤링 완료 신호를 수신하는 메시지 큐 |
| **S3 버킷** | 크롤링 결과 파일 저장소 |
| **Scheduler** | EC2에서 실행되는 Airflow 스케줄러, SQS 폴링 |
| **SqsSensor** | SQS 메시지 수신 대기 센서 |
| **후처리 DAG 트리거** | downstream task 시작 |
| **DB / DW** | 처리 결과 적재 (데이터베이스 / 데이터 웨어하우스) |
| **Lambda** *(선택)* | S3 이벤트 브릿지 역할 (선택적 구성) |

### 실행 흐름

```
SQS (완료 신호 수신)
    └─▶ S3 버킷 (크롤링 결과 저장됨)
            │
            └─▶ [S3 이벤트, 선택] ──────────────────────┐
                                                         │
EC2 — Airflow                                            │
    Scheduler (SQS 폴링)                                 │
        └─▶ SqsSensor (메시지 수신 대기)                 │
                └─▶ 후처리 DAG 트리거 ◀─────────────────┘
                        ├─▶ DB / DW (처리 결과 적재)
                        └─▶ Lambda (선택, S3 이벤트 브릿지)
```

---

## 3. 전체 데이터 흐름 요약

```
[홈서버]
Airflow Worker
    → 크롤러 실행
        → 결과를 S3에 업로드         ── (녹색 화살표: S3 결과 전달)
        → SQS에 완료 메시지 발행     ── (주황색 화살표: SQS 완료 신호)

[AWS]
SQS 수신
    → EC2 Airflow Scheduler가 SQS 폴링
        → SqsSensor가 메시지 감지
            → 후처리 DAG 트리거
                → DB/DW에 데이터 적재
                → (선택) Lambda로 S3 이벤트 처리
```

## 5. 핵심 설계 포인트

- **비동기 완료 감지**: 크롤러가 직접 후처리를 호출하지 않고, SQS 메시지를 통해 완료를 전달하여 느슨한 결합(loose coupling) 구현
- **고정 IP 활용**: 홈서버의 고정 IP를 사용해 크롤링 대상 사이트의 IP 차단을 우회하거나 일관된 출처 유지
- **선택적 Lambda 통합**: S3 이벤트를 Lambda로 처리하는 경로는 선택 사항으로, 필요에 따라 추가 가능
- **Airflow 이중 구성**: 홈서버의 Worker와 AWS EC2의 Scheduler가 역할을 분리하여 운영
